# ADR-001: Padrão de Persistência de Dados em Markdown

## Contexto
O script original (Orion Design) persistia dados em arquivos de texto plano acessados via grep/awk. Para tornar o ambiente amigável para o agente Claude e humano, precisamos de um formato auto-documentado.

## Decisão
Adotamos o uso de arquivos Markdown (.md) em `/root/dados_vps/` para toda persistência.

## Consequências
- **Prós**: Leitura humana facilitada, parsing nativo pelo Claude, integração facilitada.
- **Contras**: Requer uma camada de abstração (lib-persistence) para leitura/escrita.
