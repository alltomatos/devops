# ADR-002: Segurança de Segredos e Contexto

## Contexto
O projeto lida com credenciais, APIs e tokens. Não é aceitável persistir segredos em arquivos no repositório.

## Decisão
- Proibição de hardcode de segredos.
- Uso exclusivo de injeção de variáveis de ambiente em tempo de execução (Ephemeral Context).
- Segredos devem ser solicitados ao usuário via chat, usados e descartados.
- Auditoria: Sempre mascarar valores sensíveis no log do chat.
